## Why git

Medium buries articles behind paywalls and login walls.
Dev.to is fine but noisy.
A GitHub repo is just files — readable anywhere, diffable, linkable, forkable.

If something here is useful, you can reference it directly by file.
If something is wrong, you can open an issue or a PR.
